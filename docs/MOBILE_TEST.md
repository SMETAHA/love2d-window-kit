# Mobile/high-DPI verification

Автоматический сценарий: `tests/test_mobile_hidpi.lua`. Визуальный стенд:

```sh
love . --mobile-test
```

## Проверка на устройстве

1. Убедиться, что строка сверху показывает правильные logical size, DPI scale и orientation.
2. Провести одним пальцем внутри viewport и вывести палец за его границы — жест должен остаться у исходного окна.
3. Использовать два пальца одновременно в разных местах — индикаторы и независимые touch ID не должны перескакивать.
4. Начать drag, свернуть приложение и вернуться — старый жест должен быть отменён.
5. Повернуть устройство portrait → landscape → portrait. Floating viewport должен помещаться на экране и возвращать предпочтительный размер, когда место снова доступно.
6. Проверить scrollbar: минимальная зона 44 px, page-click, fade, hover/active на устройстве с мышью или стилусом.
7. Проверить края контента при zoom `0.5`, `1`, `2` и `3`.

## Ожидаемая модель DPI

`highdpi` и `usedpiscale` включены. Ввод уже приходит в логических координатах LÖVE; ручное умножение на `getDPIScale()` не применяется.

Аппаратная проверка остаётся обязательной перед публикацией конкретной Android/iOS сборки: desktop CI не воспроизводит драйверы touch, системное сворачивание и поворот физического устройства.
